## Система агро
Каждый моб хранит таблицу ``aggroById`` — словарь, где ключ — ``worldId`` существа, а значение — объект ``EntityAggro``. Этот объект не просто хранит число, а отслеживает **время последней угрозы** и динамически рассчитывает актуальное значение.

**1. Структура ``EntityAggro``**
| Поле     | Назначение                                                |
| -------- | --------------------------------------------------------- |
| `entity` | Ссылка на существо (игрок/питомец/моб)                    |
| `value`  | Базовое значение угрозы (на момент последнего обновления) |
| `time`   | Время последнего изменения (`Time.time`)                  |

**2. Три фазы «памяти» моба**

Система использует три временных порога:
| Период              | Прошло времени | Что происходит с агро                    |
| ------------------- | -------------- | ---------------------------------------- |
| **Активная угроза** | 0–3 секунды    | Агро равно полному `value`, без потерь   |
| **Забывание**       | 3–20 секунд    | Линейное убывание от 100% до 0%          |
| **Полное забвение** | 20+ секунд     | Моб полностью «забыл» существо, агро = 0 |

**Формула убывания**
```
если прошло > 3 сек, но < 20 сек:
    множитель = (20 - прошло_время) × 0.05882353
    текущее_агро = value × множитель
```

**Ключевой момент:** коэффициент ``0.05882353`` — это примерно ``1/17``. За 17 секунд (с 3-й до 20-й) агро падает **линейно** от 100% до 0%. На 3-й секунде множитель = 1.0, на 20-й — 0.0.

**Защита от преждевременного нуля**

Если ``value`` был положительным, но расчёт дал 0 — функция возвращает 1 (моб ещё не совсем забыл, но почти).

**3. Добавление агро — ``AddAggro``**

**Исключение для охранников**
```
если моб — Guard
   И атакующий — WTPet
   И у моба уже есть цель:
       → игнорировать
```
Охранники не переключаются на питомцев, если уже в бою. Это защита от «питомцев-танков».

**Новый агрессор**

Если существа нет в таблице — создаётся запись ``EntityAggro(entity, aggroValue)``.

**Важно:** если переданное ``aggroValue`` равно 0, система всё равно ставит **минимум** 10. Даже «нулевой» удар даёт базовую угрозу.

**Обновление существующей угрозы**
```
если value == 0 → value = 10

текущее = GetCurrentValue()   // с учётом убывания за время простоя!
this.value = текущее + новое_value
this.time = Time.time          // таймер сбрасывается
```
**Критически важно:** при новом ударе система **сначала** считает, сколько агро «сгорело» за время простоя, **потом** добавляет новое значение и **сбрасывает таймер**. Это значит, что спам-урон поддерживает высокое агро, но если перестать бить — оно быстро тухнет.

**Режим ReaperFury (ярость)**
```
если value == 0 → value = 10

текущее = GetCurrentValue()
this.value = (текущее × -1) - новое_value
this.time = Time.time
```
При активном эффекте **ReaperFury (ярость)** механика инвертируется:
- Ваше текущее агро **переворачивается в отрицательное** (× -1).
- Сверху вычитается новое значение.
- Результат: чем больше вы бьёте во время ReaperFury, тем **ниже** (отрицательнее) становится ваше агро.

**4. Полная очистка**

Удаляет все записи. Моб «забывает» всех. Вызывается при смерти, респавне, сбросе боя.

**5. Нормализация после ReaperFury (ярость)**

Когда эффект заканчивается:
```
для каждой записи:
    текущее = GetCurrentValue()
    
    если value < 0 ИЛИ текущее < 0:
        value = max(0, текущее)
        time = Time.time
```
- Всё отрицательное агро **обрезается до 0**.
- Положительное агро остаётся как есть.
- Таймер обновляется, чтобы система забывания начала отсчёт заново

**Итого**
**Агро — это не просто счётчик, а таймер.** Моб помнит угрозу **3 секунды** полностью, затем **17 секунд** линейно забывает, и через **20 секунд** стирает из памяти. Каждый новый удар обновляет таймер, но сначала «сжигает» простой. **ReaperFury (ярость)** временно переворачивает агро в отрицательное, заставляя моба игнорировать вас в пользу союзников.

## Урон от оружия (CreateDamage)
**1. Создание объекта урона**

Система инициализирует новый объект WTDamage и записывает в него название профильного навыка (profSkillName), который используется в атаке.

**2. Базовый урон (случайное значение из диапазона)**
```
baseDamage = Random(minDamage, maxDamage + 1)
basePlagueDamage = Random(minPlagueDamage, maxPlagueDamage + 1)  // если задан чумной урон
```
Урон не фиксирован — каждый удар выбирает случайное значение в пределах минимума и максимума.

**3. Проверка кастера и оружия**

Если атакующий существует и является игроком, система заглядывает в его правую руку (основной слот оружия):
- Проверяет, есть ли у оружия бонус WitchcraftBattleSkill — это влияет на зачарования позже.
- Считывает требуемый навык для оружия (skillRequired) и его уровень у игрока.
- Получает название зачарования из слота экипировки [0] (головной убор/шлем).

**4. Множители от уровня навыка владения оружием**

Если оружие требует навык, рассчитываются три коэффициента через GetValueWithSkillRequired:
| Множитель | Назначение                | Базовый штраф            |
| --------- | ------------------------- | ------------------------ |
| `num3`    | Множитель **урона**       | −50% при 0 уровне навыка |
| `num4`    | Множитель **шанса крита** | −30% при 0 уровне навыка |
| `num5`    | Множитель **силы крита**  | −15% при 0 уровне навыка |

``Чем ниже уровень навыка относительно требуемого, тем сильнее штраф. При полном соответствии множители равны 1.``

**5. Применение бонуса силы атаки кастера**

Если у кастера есть attackDamage (бонусный модификатор), он умножается на базовый урон и чумный урон:
```
damage += damage * attackDamage
plagueDamage += plagueDamage * attackDamage
```

**6. Применение навыкового множителя (num3)**

Тот самый штраф/бонус от владения оружием применяется к финальному урону:
```
damage = damage * num3
plagueDamage = plagueDamage * num3
```

**7. Зачарование**

Если на предмете есть зачарование, система считывает его параметры:
- Добавочный урон (addedDamage, addedPlagueDamage)
- Крит: шанс (addedCriticalChance) и сила (addedCriticalValue)
- Пробитие: защиты (addedIgnoreDefense) и поглощения (addedIgnoreAbsorb)
- Вампиризм (addedLifeSteal)
- Эффект: название, шанс срабатывания, длительность

Особое правило ядовитых зачарований
Если зачарование — оружейный яд (WeaponPoisonEnchantment) и у игрока нет пассивки WitchcraftBattleSkill, то все бонусы зачарования режутся на 20% (умножаются на 0.8).

**8. Расчёт критического удара**
Финальные параметры крита формируются так:
```
criticalValue = baseCriticalValue * num5          // от навыка
criticalChance = baseCriticalChance * num4        // от навыка
```
Дополнительно прибавляются бонусы кастера:
- ``EntityCritValueAddPercent`` — к силе крита
- ``EntityCritChanceAddPercent`` — к шансу крита

**Удар «ChargedStrike»**
Если на кастере висит эффект ChargedStrike:
- Шанс крита становится 100%,
- Эффект сразу расходуется.

**Ролл на крит**
```
if (Random.value <= criticalChance) {
    isCritical = true
    damage = damage * criticalValue
    plagueDamage = plagueDamage * criticalValue
}
```

**9. Сборка финального объекта урона**

В WTDamage записываются все рассчитанные и исходные параметры:
- damage / plagueDamage — итоговые значения
- isCritical — был ли крит
- addedDamage / addedPlagueDamage — бонус от зачарования
- ignoreDefense, ignoreAbsorb, ignoreBlock, ignoreDodge — пробитие защитных статов
- addedLifeSteal — вампиризм
- addedEffects — эффекты от зачарования
- effects — базовые эффекты атаки
- abilityName — название способности (для отладки)

**10. Эффекты сложности подземелий**

Если кастер — моб в данже, в зависимости от уровня сложности добавляются дополнительные эффекты:
- Hard — добавляются effectsHard
- Nightmare — добавляются effectsNightmare

**11. Процентные бонусы урона к типам существ (только для игроков)**

Если кастер — игрок, в объект урона копируются его специализированные бонусы:
| Поле                    | Назначение            |
| ----------------------- | --------------------- |
| `toMobsPercent`         | Урон к обычным мобам  |
| `toMobsBossPercent`     | Урон к боссам         |
| `toAnimalsPercent`      | Урон к животным       |
| `toInsectsPercent`      | Урон к насекомым      |
| `toColdBloodedPercent`  | Урон к хладнокровным  |
| `toSkeletonsPercent`    | Урон к скелетам       |
| `toPlaguePercent`       | Урон к чумным         |
| `toBirdAndHarpyPercent` | Урон к птицам/гарпиям |
| `toPlayersPercent`      | Урон к игрокам        |

Эти модификаторы применяются уже при нанесении урона цели, в зависимости от её типа.

**Итоговая формула**

Урон = (случайный базовый × сила атаки кастера × штраф от навыка + бонусы зачарования) × крит (если выпал). При этом зачарования яда слабее без ведьмовского скилла, а в подземельях мобы получают дополнительные эффекты в зависимости от сложности.

## Урон от навыков (DoAbilityOnDamage)

**1. Усилитель болта — «чем дальше, тем больнее»**
```
урон += урон × 0.1 × расстояние
чумной_урон += чумной_урон × 0.1 × расстояние
```
- Урон увеличивается пропорционально расстоянию между кастером и целью.
- За каждую единицу дистанции — +10% к урону.
- Чем дальше стреляешь, тем сильнее прилетает.

**2. Смертельный удар — добивание по процентам от здоровья**
| Параметр          | Базовое значение | При ударе сзади | Против босса |
| ----------------- | ---------------- | --------------- | ------------ |
| Шанс срабатывания | 6%               | 12% (×2)        | 1.8% (×0.3)  |
| Урон от max HP    | 25%              | 25%             | 7.5% (×0.3)  |

**Механика:**
- Бросается кубик. Если выпал шанс — к урону добавляется процент от максимального здоровья цели.
- Дополнительный урон распределяется между обычным и чумным уроном пропорционально их соотношению.
- Удар помечается как критический (isCritical = true).
``Против боссов способность сильно ослаблена: шанс и урон падают до 30%.``

**3. Костолом**
```
если цель под эффектом "Stun":
    если удар ещё не крит:
        урон = базовый_урон × сила_крита
        чумной_урон = базовый_чумной × сила_крита
    удар = крит
```
- Работает только на оглушённых целях.
- Если удар уже был критом — просто подтверждает флаг.
- Если не был — превращает обычный удар в крит, домножая на ``criticalValue``.

**4. Точно в глаз**
| Условие                | Урон                             | Крит?              |
| ---------------------- | -------------------------------- | ------------------ |
| Удар **сзади**         | Базовый урон (без модификаторов) | Нет                |
| Удар **спереди/сбоку** | Базовый урон × 2                 | Да (принудительно) |

- Способность **поощряет лобовую атаку**: в лицо урон ×2 и гарантированный крит.
- В спину — наоборот, урон сбрасывается до «чистого» базового значения.

**5. Удар в спину**
| Условие           | Урон                                    | Крит? | Эффекты      |
| ----------------- | --------------------------------------- | ----- | ------------ |
| Удар **сзади**    | Базовый урон × 5                        | Да    | Как есть     |
| Удар **не сзади** | Базовый урон (сброс всех модификаторов) | Нет   | **Обнулены** |

- Работает зеркально ``Точно в глаз``: награда за удар в спину — ×5 урона.
- Если промахнулся по позиции — урон падает до голого базового, крит снимается, все эффекты атаки удаляются.

**6. Напополам**
```
множитель = 1 − процент_здоровья_цели    // 0..1
урон += урон × множитель
чумной_урон += чумной_урон × множитель

если у цели < 50% HP и удар не крит:
    урон = урон × сила_крита
    чумной_урон = чумной_урон × сила_крита
    удар = крит
```
- Урон **увеличивается пропорционально потерянному здоровью** цели.
- У цели с 30% HP — бонус +70% к урону.
- Если цель **ниже половины здоровья** и удар не был критом — он превращается в крит с полной силой крита.

**Общая логика**
| Способность     | Тип          | Ключевая механика                 |
| --------------- | ------------ | --------------------------------- |
| **Усилитель болта** | Дальнобойная | Урон растёт с дистанцией          |
| **Смертельный удар**   | Шансовый     | Процент от max HP цели            |
| **Костолом** | Контроль     | Крит по оглушённым                |
| **Точно в глаз**  | Позиционная  | Бонус за лобовую атаку            |
| **Удар в спину**    | Позиционная  | Бонус за удар в спину             |
| **Напополам**     | Добивание    | Чем меньше HP у цели, тем больнее |

**Важно:** если ``abilityName`` не совпал ни с одним из шести вариантов — функция ничего не делает, урон проходит «как есть».

## Атака питомцем (GetPetAttack)

**1. Базовый урон питомца**
```
minDamage = pet.minDamage + pet.addDamage
maxDamage = pet.maxDamage + pet.addDamage
```
- Урон питомца состоит из его **базового диапазона** (minDamage / maxDamage) плюс **добавочный урон** (addDamage).
- addDamage — это бонус от уровня, экипировки или талантов хозяина.

**2. Модификаторы от эффектов**

**Усиление атаки**
```
если эффект активен:
    урон × 1.1  (+10%)
```
- Если на питомце висит эффект ``Усиление атаки`` (вероятно, от таланта или способности хозяина), его урон повышается на **10%**.

**Адаптация**
```
если эффект активен:
    урон × 0.75  (−25%)
```
- Эффект BatAdaptation снижает урон на **25%**.
- Атаки игнорируют уворот, блок, парирование и поглощение.

**3. Эффекты атаки питомца**

В атаку добавляются все эффекты, привязанные к питомцу (отравление, оглушение, замедление и т.д.).

**4. Модификаторы крита и пробития**
| Параметр         | Откуда берётся                             |
| ---------------- | ------------------------------------------ |
| `criticalChance` | Базовое значение + `pet.addCriticalChance` |
| `criticalValue`  | Базовое значение + `pet.addCriticalValue`  |
| `ignoreDefense`  | Базовое значение + `pet.addIgnoreDefense`  |
| `ignoreBlock`    | Базовое значение + `pet.addIgnoreBlock`    |
| `ignoreDodge`    | Базовое значение + `pet.addIgnoreDodge`    |

Питомец может иметь собственные бонусы к криту и пробитию защитных характеристик цели — они **прибавляются** к базовым значениям атаки.

**5. Шанс сброса с транспорта**
Шанс сбить цель с лошади/транспорта берётся из исходного шаблона атаки, а не от питомца. То есть питомец наследует эту характеристику от типа атаки, а не от своих статов.

**6. Заряженный удар**
```
если на питомце эффект "ChargedStrike":
    criticalChance = 1.0  (100%)
    эффект расходуется
```
- Гарантированный крит.
- Эффект снимается сразу после применения — это одноразовый бонус.

**Итоговая формула**

Урон питомца = (базовый_урон_питомца + addDamage) × 1.1 (если **Усиление атаки**) × 0.75 (если **Привыкание**), плюс добавочные крит-шанс/сила и пробитие защит от статов питомца, плюс его личные эффекты на атаку.

## Начисление PVP рейтинга (ApplyPvpRatingOnKill)

**1. Проверка условий (фильтрация)**
Перед расчётом очков система проверяет ряд условий. Если хотя бы одно не выполняется — рейтинг не изменяется:
- Режим PvP — на PvE-серверах рейтинг за убийства не начисляется.
- Отсутствие эффекта StartBoostXp — если у убийцы или жертвы активен стартовый буст опыта, рейтинг не меняется.
- Защита жертвы ≥ 70% — если показатель защиты жертвы ниже 70%, очки не изменяются.
- Рейтинг жертвы > 50 — слишком слабых игроков (рейтинг 50 и ниже) нельзя «фармить» для набора очков.

**2. Сохранение текущих значений**
Система запоминает текущий рейтинг обоих участников:
- num — рейтинг жертвы (до изменения)
- num2 — рейтинг убийцы (до изменения)

**3. Расчёт ожидаемого результата (формула Эло)**
Используется классическая Elo-формула с основанием 10:
```
num3 = 1 / (1 + 10^((рейтинг_жертвы − рейтинг_убийцы) / 400))
```

**4. Расчёт дельты (потенциальное изменение рейтинга)**
```
num4 = round(24 × (1 − num3))
```
- K-фактор = 24 — максимальное количество очков, которое теоретически может перейти от жертвы к убийце.
- Чем сильнее убийца по рейтингу, тем меньше num4 (за победу над слабым дают мало).
- Чем слабее убийца, тем больше num4 (за победу над сильным дают много).
- Если num4 округляется до 0, расчёт прерывается — изменений не происходит.

**5. Применение нижнего порога рейтинга**
```
num5 = max(50, рейтинг_жертвы − num4)
```
- Новый рейтинг жертвы не может опуститься ниже 50.
- num5 — это фактический рейтинг жертвы после поражения.

**6. Финальная дельта (сколько реально перейдёт очков)**
```
num6 = рейтинг_жертвы − num5
```
- Из-за порога в 50 очков реальная потеря может быть меньше рассчитанной num4.
- Если num6 равно 0, изменений не происходит (например, жертва уже на дне рейтинга).

**7. Применение изменений**
```
жертва.рейтинг  = num5          (теряет num6 очков)
убийца.рейтинг   = num2 + num6   (получает num6 очков)
```
- Очки передаются напрямую от жертвы к убийце.
- Общая сумма рейтинга в системе остаётся неизменной (если не сработал порог 50).
